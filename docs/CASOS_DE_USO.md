REGISTRO DE CASOS DE USO DE LOS DISTINTOS ACTORES DEL SISTEMA
---
# ACTORES:
1. Donante
2. Administrador del Hospital
3. Super Admin
---
# CASOS DE USO
### Para el Donante:
- CU1. Como donante quiero tener una forma de registrarme y crear una cuenta para entrar al sistema.
- CU2. Como usuario quiero ver las solicitudes de dadores para postularme.
- CU3. Como potencial donante quiero saber si estoy apto para donar.- CU4. Como usuario quiero tener acceso a los datos de la solicitud para poder ir a donar.
- CU5. Como usuario quiero tener posibilidad de cerrar sesión para salir del sistema.
- CU6. Como usuario quiero tener posibilidad de eliminar mi cuenta para darme de baja del sistema.- CU7. Como usuario quiero tener posibilidad de editar mis datos en el perfil para modificarlos.
### Para el Hemo-Admin:
- CU8.  Como hemo-admin quiero crear una petición de urgencia o solicitud detallando grupo, factor, cantidad de donantes, fecha de creación para notificar el sistema.
- CU9. Como hemo-admin quiero poder editar o eliminar una petición para modificarla o darla de baja del sistema.
- CU10. Como hemo-admin quiero poder marcar como completada una petición  para darla por confirmarla en el sistema.
- CU11. Como hemo-admin quiero poder generar informes sobre cuanta gente vino a donar, cuantos de que factor y tipo de sangre, cuantas solicitudes se completaron para tener retroalimentación.
### Para el Administrador:
- CU12. Como administrador quiero poder crear administradores de centros de hemoterapia para que estén habilitados a ingresar al sistema.
- CU13. Como administrador quiero poder editar o eliminar administradores de centros de hemoterapia y sus datos para darlos de baja del sistema.
